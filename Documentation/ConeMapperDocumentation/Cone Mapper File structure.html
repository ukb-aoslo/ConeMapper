Cone Mapper data file structure

The Cone Mapper data file consists of the next fields:
  • conecount - total number of annotated cones
  • I - the annotated retinal image
  • conelocs - the cone locations in format [x, y, detectIndex]
  • euclidianNCones - the struct with the density information of the retinal image based on average area of N nearest cones
  • nnd_mean - the struct with the density information of the retinal image based on average distance to nearest neighbours
  • yellotsRings - the struct with the density information of the retinal image based on frequency analysis of the image
The detectIndex can have the next values:
  • 1 - means that cone was detected by some algorithm (for example CNN)
  • 2 - means that cone was detected manually by user
  • Other values are deviance and has no meaning
The common fields for all density structs:
  • DensityMatrix - the density matrix itself. These values are used in Cone Mapper app to represent the density map. Units:
  • PCD_cppa - peak cone density (PCD) value
  • MinDensity_cppa - minimum density value
  • PCD_loc - coordinates of PCD in the density matrix
  • CDC20_density - cone density centroid (CDC) density value
  • CDC20_loc - CDC location
  • Stats2 - raw measured statistic structure. Has only one field inside: WeightedCentroid, which is the CDC location in float coordinates
The fields of euclidianNCones:
  • Vorocones - the cone locations which is used for density calculation
  • ImageHeight - the height of the retinal image
  • ImageWidth - the width of the retinal image
  • NumOfNearestCones - the number of nearest cones, which is used for density calculation
  • GoodPointsEdge - the polygon. Outside of the polygon density values consider to be approximated towards the center.
  • GoodPointsMap - the mask, representing "non-approximated" density values
The fields of nnd_mean:
  • Vorocones - the cone locations which is used for density calculation
  • ImageHeight - the height of the retinal image
  • ImageWidth - the width of the retinal image
  • AvgDistancesToNeighbors - the average distance between 2 cones in pixels, for each cone location.
  • AvgDistanceMap - the interpolated average distance from cone location into full map.
  • StdDistancesToNeighbors - the std distance between 2 cones in pixels, for each cone location.
  • StdDistanceMap - the interpolated std distance from cone location into full map.
  • MinDistancesToNeighbors - the min distance to neighbour cone in pixels, for each cone location.
  • MinDistanceMap - the interpolated min distance from cone location into full map.
  • MaxDistancesToNeighbors - the max distance to neighbour cone in pixels, for each cone location.
  • MaxDistanceMap - the interpolated max distance from cone location into full map.
  • DensityMatrixConesPerAreaNotFiltered - the density map, not smoothed. Cones per hexagonal area:
The fields of yellotsRings:
  • SourceImage - the retinal image, which is used for density calculation
  • avgPixelSpac - the average spacing of the image
  • interpedSpacMap - the spacing map of the input image (in pixel spacing)
  • interpedConfMap - the confidence map of the input image
  • sumMap - the map corresponding to the amount of ROI overlap across the output map
  • imBox - the bounding region of valid (nonzero, NaN, or Inf) pixels
