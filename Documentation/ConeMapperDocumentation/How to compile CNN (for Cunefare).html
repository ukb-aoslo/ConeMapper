How to compile CNN (for Cunefare)

Introduction

This instruction shows the way and problems with compilation on Windows OS.
To make the compilation we need MS Visual Studio, supported by your current version of Matlab.
List of supported versions can be found here: https://nl.mathworks.com/support/requirements/supported-compilers.html
To make the compilation of GPU version we also need CUDA Toolkit.
The base instruction you can find here: https://www.vlfeat.org/matconvnet/install/
All steps listed here was tested with Matlab R2022b, MS Visual Studio 2022, CUDA Toolkit v11.7

Base steps

First step is to install MS Visual Studio and to show Matlab that we have a C++ compiler.
Use next command in Matlab command line mex -setup C++.
Next, we should decide which type of CNN we want to compile (gpu / cpu).
If you don't have NVIDIA graphic card then you cannot use CUDA Toolkit, and you cannot compile GPU version.

CPU compilation

To compile CPU version you should do next commands in Matlab command line:
  • cd <MatConvNet> - open MatConvNet directory as your working directory
  • addpath matlab - add `./matlab` path to Matlab search path
  • vl_compilenn - compile cpu version
If all compiles ok, then you will see no errors.
The common error that you can face is
Error using vl_compilenn>check_clpath (line 591)
Unable to find cl.exe
It is described here: https://stackoverflow.com/questions/40226354/matconvnet-error-cl-exe-not-found
In short, you need to
  • Open the Start Search, type in "env", and choose "Edit the system environment variables".
  • Click the "Environment Variables…" button.
  • Find variable "Path" in "System Variables".
  • Add path to "cl.exe". It should be "<path to Visual Studio>\2019\Community\VC\Tools\MSVC\<version>\bin\Hostx64\x64". For example: "C:\Program Files (x86)\Microsoft Visual Studio\2019\Community\VC\Tools\MSVC\14.26.28801\bin\Hostx64\x64".
  • After editing "Environment Variables…" you should fully close your Matlab and open it again.
After issue is solved you can do again the compilation.

GPU compilation

To compile GPU version you should do next commands in Matlab command line:
  • cd <MatConvNet> - open MatConvNet directory as your working directory
  • addpath matlab - add `./matlab` path to Matlab search path
  • vl_compilenn('enableGpu', true, 'cudaRoot', '<path_to_CUDA>', 'verbose', 1, 'Debug', true) - compile GPU version
For example: vl_compilenn('enableGpu', true, 'cudaRoot', 'C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.1', 'verbose', 1, 'Debug', true)
Note: 'Debug' option and 'verbose' option only shows more output in console. It is not necessary for compilation.
If all compiles ok, then you will see no errors.
You can face the issue which is described in "CPU compilation" topic (Unable to find cl.exe). Also, you can face next problems:

First one:

You should check the path --compiler-bindir=. If there is no folder, then just create it. For me the paths were next (for different Visual Studio versions):
  • C:\Program Files (x86)\Microsoft Visual Studio\2019\Community\VC\bin
  • C:\Program Files (x86)\Microsoft Visual Studio\2017\Community\VC\bin
Detailed discussion you can find here: https://github.com/vlfeat/matconvnet/issues/1062

Second one:

Solution.
In vl_compilenn.m modify next lines:
line 620:
args = horzcat({'-outdir', mex_dir}, ...
flags.base, flags.mexlink, ...
'-R2018a',...
{['LDFLAGS=$LDFLAGS ' strjoin(flags.mexlink_ldflags)]}, ...
{['LDOPTIMFLAGS=$LDOPTIMFLAGS ' strjoin(flags.mexlink_ldoptimflags)]}, ...
{['LINKLIBS=' strjoin(flags.mexlink_linklibs) ' $LINKLIBS']}, ...
objs) ;
line 359:
flags.mexlink = {'-lmwblas'};
Detailed discussion you can find here: https://github.com/vlfeat/matconvnet/issues/1143

What environment variables should be set

  • CUDA_PATH - should lead to the main folder of the CUDA toolkit
  • Path - in path variable should be set paths to <CUDA_ROOT>\bin and <CUDA_ROOT>\libnvvp
